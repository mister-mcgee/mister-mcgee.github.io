## Bugs v Errors

<Swbat>Explain the difference between errors and bugs.</Swbat>

An *error* is the result of a known edge-case in a program or software system. Errors can be detected by the system and either handled gracefully (known as recovering) or the system can halt its execution (known as crashing). Useful errors will contain information about *what* went wrong and *where* it went wrong.


**An error is not a bug.**

An *error* occurs when a *program* receives a result that violates one or more expectations. 

A *bug* occurs when a *programmer* receives a result that violates one or more expectations.

For many new programmers, errors are both intimidating and a source of frustration. What I want you to remember is that errors are *tools*.

A bug is the result of a programmer's incomplete logic or incorrect assumptions. Some bugs may produce errors, but more often than not a bug will produce unexpected behavior while the program is running.
